# Implementation Notes

- Exact requested model aliases were not all callable as separate workers. Bounded subagent probes used `gpt-5.3-codex-spark`; broad probes failed with context limits.
- The repo keeps product logic in `brainmap-cli`; model bytes are downloaded and checksum-verified by `build.rs`, then embedded into the binary.
- Real `minishlab/potion-base-8M` assets are used for local embedding through `model2vec-rs` with `local-only` and `fancy-regex` features.
- Encrypted export/verify/restore works with age x25519 recipients and identity files.
- MCP stdio JSON-RPC server is implemented for `initialize`, `ping`, `tools/list`, and `tools/call`. It supports Content-Length framed input and JSONL smoke input. Only allowlisted Brainmap tools are exposed; no shell tool exists.
- The web UI uses a small native local HTTP server and static assets generated by Rust. No write endpoints exist.
- AgentMemory export mode parses JSON string leaves, keeps decision-like signals, and discards code/project chronology/secret-like strings.
- Autopilot promotion is guarded: conservative requires at least 30 decisions, fewer than 2 serious mismatches, and zero privacy/hard-rule violations; balanced requires at least 100 decisions and zero false proceeds in MVP; aggressive is denied.
- Verified performance on this container: gate 9 ms, capture 1 ms, FTS 0 ms from `brainmap bench --vault ./tmp/BrainMap`.
- Security tools were installed and run:
  - `cargo audit` passed with allowed unmaintained transitive warnings: `RUSTSEC-2024-0436` (`paste` via `model2vec-rs -> tokenizers`), `RUSTSEC-2025-0119` (`number_prefix` via `model2vec-rs -> tokenizers -> indicatif`), and `RUSTSEC-2026-0173` (`proc-macro-error2` via `age -> i18n-embed-fl`).
  - `cargo deny check` passed with duplicate and license metadata warnings.
  - `cargo cyclonedx --format json --override-filename brainmap` produced `crates/brainmap-cli/brainmap.json`.
- Vector search uses precomputed SQLite `vector_embeddings` and an in-process cosine scan instead of loading a SQLite vector extension. This avoids extension loading while keeping retrieval local and embedded for MVP scale.
- Encrypted CLI smoke:
  - `brainmap export --mode portable --encrypt --recipient age1... --vault ./tmp/BrainMap --out ./tmp/encrypted.brainmap.tar.zst.age`
  - `brainmap verify-export ./tmp/encrypted.brainmap.tar.zst.age --identity ./tmp/age-identity.txt`
  - `brainmap restore --file ./tmp/encrypted.brainmap.tar.zst.age --identity ./tmp/age-identity.txt --to ./tmp/EncryptedRestored`
